## Product decisions that MUST NOT be regressed

1. **FB follow model (2026-06-12, supersedes relationship-separation §2.3/D3):**
   users AND pages are followable; friendship accept auto-creates mutual
   follows (graph-service `AcceptConnectionRequest`). Never re-add the
   users-not-followable guard. Friend-requesting a page is still rejected.
2. **Communities feature is disabled, not deleted** — entry points commented
   (sidebar/search/notifications/mobile router imports). Groups is the kept
   product, rebranded **"MySpace"** (display only; routes stay `/groups`).
3. **Color rules:** dark/white chrome everywhere, only icons carry accents;
   single orange element = the VC logo in the header. Never change palette
   without permission.
4. **No new URL routes without permission.** In-page state + query params
   (e.g. `/groups?space=<id>`) are the pattern.

## Current MySpace (web) feature state — all live & pushed

- `/groups` = three-pane: left SpaceMenu rail (My Feed / Discover / My
  Spaces / Invites + joined list w/ green selection highlight), middle
  content, right Sponsored placeholder (hidden while a space is open).
- `/groups/<id-or-handle>` redirects to `/groups?space=...` — one layout.
- Space view: cover → single-line header (avatar · name · privacy + members
  + 18+ chip · Create Post / Invite / Joined) → About / Discussion / People /
  Media / Rules / Manage tabs.
- Create space: one screen, Reddit-style Public/Restricted/Private +
  Mature 18+ toggle (`groups.is_mature`, migration 010), live preview,
  lands selected on MySpace.
- Manage → `/groups/<id>/settings`: FB-style console (left menu w/ badges:
  Space details / Member requests / Pending posts / Banned / Rules; right
  editor incl. cover+avatar upload, danger zone).
- People tab CTA matrix (DB-driven via `/v1/graph/relationships/batch`,
  which now returns `is_connection`): page/creator → Follow toggle only;
  friend → Following + Message (floating ChatWindow dock); pending →
  Requested/Respond; stranger → Add friend. Single "Members (n)" heading,
  admins first, no Member badges.
- Media tab: real thumbnails from v2 `group_posts.attachments`
  (Photos/Videos sub-tabs). Backend `ListGroupMedia` rewritten.
- Discover: per-viewer scored (`friends-in-space > category > location >
  popularity`) with reasons on cards; joined/hidden excluded.
- Aggregated home: `GET /v1/groups/feed` (membership join) powers My Feed;
  `GET /v1/groups/invites/my` powers Invites.
- Home page: People-you-may-know strip inline in feed (after 3rd + 15th
  post); right rail rotates Trending / PostTube / Events / News cards.
- Login page: restyled monochrome, OAuth buttons removed.
