01

Authentication & Identity

Secure, multi-factor authentication with OTP, JWT tokens, and anomaly detection across 3 dedicated services.

🔐 Phone OTP Authentication

Primary login via OTP sent to phone number. Rate limited to 5 OTPs per phone per 10 minutes.

  • OTP generation, delivery, and verification
  • Redis-backed sliding window rate limiting (INCR + EXPIRE)
  • Configurable OTP expiry and retry limits
Web Mobile API

🔑 Email + Password Login

Secure password-based authentication with strict policy enforcement.

  • Password policy: 8+ chars, uppercase, digit, special character
  • Login rate limit: 10/IP/15min + 5/identifier/15min
  • Forgot password flow with email verification
Web Mobile

🛡️ JWT Token Management

Stateless authentication with access + refresh token pairs and session tracking.

  • Short-lived access tokens + long-lived refresh tokens
  • Active session management with revocation
  • X-User-Id header injection by API Gateway

🕵️ Login Anomaly Detection

Detects suspicious login patterns using IP-based heuristics and emits security events.

  • Redis last_ip:{userID} tracking
  • Emits user.login_anomaly Kafka event on new IP
  • Foundation for future geo-fencing and device fingerprinting

📧 Email Verification & 2FA

Multi-step email verification and two-factor authentication setup.

  • Email verification link flow
  • 2FA setup page at /auth/2fa
  • OAuth callback handling at /auth/callback

📦 GDPR Data Export

Users can export all their personal data in compliance with data protection regulations.

  • GET /v1/auth/data-export endpoint
  • Downloads consolidated personal data archive
  • UI: "Download My Data" button in Settings

Infrastructure & Platform

API gateway, Kafka event bus, observability stack, internal service auth, and Docker orchestration.

🌐 API Gateway

Single entry point for all client requests with JWT validation, routing, and header injection.

  • Port 8080 — routes to all backend services
  • JWT validation and X-User-Id header injection
  • Request ID and tracing propagation

📨 Kafka Event Bus (Redpanda)

70+ event types with reliable delivery, deduplication, and dead letter queues.

  • EventEnvelope struct with event_id, event_type, payload, occurred_at
  • Consumer groups per service with Redis-backed deduplication
  • Dead letter queue (DLQ) for failed messages
  • Transactional outbox pattern for reliable publishing

🔭 Observability

  • Structured logging with log/slog across all services
  • OpenTelemetry tracing stub with middleware
  • Prometheus metrics collection
  • Health check helpers for Postgres, Redis, ScyllaDB

🔑 Internal Service Auth

X-Internal-Service-Key header enforced by shared middleware on all service-to-service calls.

🐳 Docker Compose

Full-stack orchestration with infrastructure and application services.

  • Infrastructure-only mode: docker-compose.infra.yml
  • Full stack: docker-compose.yml
  • run-local.sh for running specific services against Docker infra
  • Multi-stage Alpine Docker builds for all services

Dating App — Trust-First Platform

A standalone trust-first dating platform built as a Go modular monolith. Separate repo (dating-app/), reuses AtPost tech stack patterns. Designed for authenticity over swiping.

💜 OTP Authentication & JWT Sessions

Phone-based OTP login with JWT access + refresh token pairs. Designed for a mobile-first dating experience.

  • 6-digit OTP via Redis with 5-minute TTL
  • Rate limited: 5 OTPs/phone/10min, 10 verifications/IP/15min
  • Configurable bypass code for dev/testing environments
  • Refresh token rotation stored in Redis
Go/Gin Redis OTP JWT HS256

👤 Profile Onboarding & Completion Tracking

Rich dating profiles with 25+ fields, automatic completion percentage, and strict 18+ age validation.

  • Profile fields: name, DOB, gender, looking_for, bio, occupation, education, city, location, religion, lifestyle
  • Automatic completion % calculation (weighted per field)
  • 18+ age enforcement at service layer
  • Onboarding status machine: started → auth_complete → profile_started → ready
Go/Gin PostgreSQL Gin Binding Validation

🎯 Preferences & Dealbreakers

Granular match preferences with age range, distance, gender, intent, lifestyle filters, and hard dealbreakers.

  • Min/max age (floor: 18), distance in km, gender preference
  • Relationship intent: long_term, marriage, casual, figuring_out
  • Lifestyle preferences: smoking, drinking, wants_children, religion
  • JSON dealbreakers array for hard-filter exclusions

📸 Photo Upload (MinIO Presigned URLs)

Secure presigned URL upload flow for profile photos. Max 6 photos per user with auto-primary assignment.

  • Init upload → presigned PUT URL → complete upload (3-step flow)
  • Allowed types: JPEG, PNG, WebP (max 20MB)
  • Auto-approve moderation for MVP; visibility: public/blurred/private/reveal_after_match
  • Primary photo auto-set on first upload
Go/Gin MinIO S3 Presigned URLs

🔍 Discovery Feed & Weighted Scoring

Intelligent candidate ranking using a multi-factor compatibility score. Hard filters exclude blocked, decided, and banned users.

  • Score formula: 20% age_fit + 15% distance_fit + 20% intent_fit + 15% lifestyle_fit + 10% trust + 10% profile_quality + 10% freshness
  • Haversine distance calculation for geo-matching
  • Hard exclusions: blocked (bidirectional), already decided, banned/suspended
  • Gender filter, over-fetch + sort for optimal page quality
Go Scoring Engine PostgreSQL Haversine

💕 Like/Match System & Decisions

Like, pass, or super_like. Mutual likes auto-create a match and conversation instantly.

  • Decisions: like, pass, super_like (extensible: reveal_request, focus_unlock)
  • Mutual like detection → instant match creation
  • UUID ordering: always store lower UUID as user_a_id for consistency
  • Match response includes match_id + conversation_id for immediate chat
Go/Gin PostgreSQL UUID Ordering

💬 Chat (REST MVP)

Direct messaging between matched users. Auto-created on match, with cursor pagination and membership checks.

  • Auto-conversation creation when match happens
  • Message types: text, image, icebreaker
  • Membership verification on every read/write
  • Conversation closes on unmatch or block
Go/Gin PostgreSQL

🛡️ Reports, Blocks & Safety

Full safety system with categorized reports, bidirectional blocks, safety events, and auto-priority escalation.

  • Report categories: fake_profile, scam, harassment, underage, violence, etc.
  • Urgent auto-escalation for underage/violence/sexual_content reports
  • Block → close active match + close conversation (atomic)
  • Safety events table with severity tracking
Go/Gin PostgreSQL Audit Logs

👑 Admin Moderation Queue

Role-based admin panel (API) with report queue, user lookup, suspend/ban actions — all audit-logged.

  • Role middleware: admin, moderator — DB-backed role loading
  • Priority-sorted moderation queue (urgent → high → normal)
  • Resolution actions: dismissed, warned, suspended, banned
  • Every admin action → audit_logs table for compliance
Go/Gin PostgreSQL Role-Based Access

🏗️ Architecture & Schema

Modular monolith with clean domain boundaries. 23 PostgreSQL tables, idempotent schema bootstrap, seed data.

  • Single binary, 7 domain modules: auth, profiles, media, discovery, matching, chat, moderation
  • 23 tables + 18 indexes — runs CREATE TABLE IF NOT EXISTS on every startup
  • Seed data: 6 profile prompts, 3 subscription plans (Free/Plus/Premium)
  • Module dependency graph: services take interfaces, main.go wires concrete types
  • 14 unit tests covering scoring, OTP, UUID ordering, API responses, profile completion
